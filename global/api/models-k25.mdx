kimi-k2.5 is Kimi's most versatile model to date, featuring a native multimodal architecture that supports both visual and text input, thinking and non-thinking modes, and dialogue and Agent tasks. Context window: 256k.

## Parameter Constraints

The following parameters have special restrictions on kimi-k2.5, different from other models:

| Parameter | kimi-k2.5 Value | Note |
|-----------|----------------|------|
| `temperature` | — | **Cannot be modified** |
| `top_p` | Fixed at 0.95 | **Cannot be modified**. Setting other values returns an error. |
| `n` | Fixed at 1 | **Cannot be modified** |
| `presence_penalty` | Fixed at 0 | **Cannot be modified** |
| `frequency_penalty` | Fixed at 0 | **Cannot be modified** |

## thinking Parameter

kimi-k2.5 supports the `thinking` parameter to control whether deep thinking is enabled. Thinking is **enabled by default**.

### Enable Thinking (Default)

```python
from openai import OpenAI

client = OpenAI(
    api_key="$MOONSHOT_API_KEY",
    base_url="https://api.moonshot.ai/v1",
)

# Thinking is enabled by default, no need to specify
completion = client.chat.completions.create(
    model="kimi-k2.5",
    messages=[
        {"role": "user", "content": "Explain why 1+1=2."}
    ],
    max_tokens=1024*32,
    stream=True,
)
```

### Disable Thinking

Since the OpenAI SDK doesn't have a native `thinking` parameter, use `extra_body`:

<CodeGroup>
```python Python
completion = client.chat.completions.create(
    model="kimi-k2.5",
    messages=[
        {"role": "user", "content": "Hello"}
    ],
    extra_body={
        "thinking": {"type": "disabled"}
    },
    max_tokens=1024*32,
)
```

```bash cURL
curl https://api.moonshot.ai/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $MOONSHOT_API_KEY" \
  -d '{
    "model": "kimi-k2.5",
    "messages": [
      {"role": "user", "content": "Hello"}
    ],
    "thinking": {"type": "disabled"}
  }'
```
</CodeGroup>
